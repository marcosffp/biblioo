# Gerência de Projetos — Biblioo

Este diretório contém os artefatos formais de gerência de projetos produzidos ao longo do desenvolvimento do Biblioo, conforme exigido pela disciplina de Engenharia de Software da PUC Minas.

---

## Documentos

### Termo de Abertura do Projeto (TAP)

O TAP formaliza o início do projeto e autoriza sua execução. Disponível em dois formatos:

| Formato | Arquivo |
|---|---|
| Markdown (editável) | [termo_de_abertura_do_projeto.md](./termo_de_abertura_do_projeto.md) |
| PDF (versão final) | [termo-de-abertura-do-projeto.pdf](./termo-de-abertura-do-projeto.pdf) |

**Conteúdo do TAP:**
- **Identificação:** nome do projeto, gerente (Carlos José Figueiredo), tipo (novo produto)
- **Objetivo:** plataforma de comunidade literária web e mobile para leitores ativos brasileiros
- **Escopo:** registro de leituras, estantes virtuais, recomendações, interação social, DNA Literário
- **Contra-escopo:** sem marketplace, sem leitura de e-books, sem trocas de livros físicos
- **Estimativa de prazo:** 23/02/2026 a 06/07/2026 (1 920 horas)
- **Estimativa de custo:** R$ 78.100,00 (recursos humanos + hardware + hospedagem)
- **Partes interessadas:** 6 integrantes + 3 professores orientadores

---

### Estrutura Analítica do Projeto (EAP)

A EAP decompõe o escopo do projeto em entregas hierárquicas, facilitando o planejamento e o acompanhamento das atividades.

| Formato | Arquivo | Descrição |
|---|---|---|
| PDF (visão hierárquica) | [EAP - Biblioo.pdf](./EAP%20-%20Biblioo.pdf) | EAP completa com todos os níveis de entrega |
| PDF (árvore compacta) | [EAP - Biblioo - Tree.pdf](./EAP%20-%20Biblioo%20-%20Tree.pdf) | Visualização em árvore para visão rápida do escopo |

**Estrutura de alto nível da EAP:**

| Pacote | Entregas principais |
|---|---|
| 1. Gerenciamento do Projeto | TAP, EAP, atas de kickoff e sprints |
| 2. Descoberta e Definição do Produto | Módulos de usuário, livros, comunidades, recomendações, feed e DNA Literário |
| 3. Testes e Qualidade | Testes unitários, integração, front-end, mobile, segurança e avaliação heurística |
| 4. Implantação | Preparação do ambiente, publicação da API e entrega ao serviço web |
